# FITOVA v6 – خطة التنفيذ التفصيلية

> **الحالة:** جاهز للتنفيذ  
> **المشروع:** FITOVA – Next.js + Supabase  
> **المرجع:** `v6.plan.md`

---

## ملخص تنفيذي

يتمحور v6 حول إعادة هيكلة جوهرية تشمل:
1. **Navbar** بقائمة Mega Menu كاملة مع Slider وتسلسل هرمي جديد للفئات
2. **نظام فلترة موحد** يخدم Outfits / Deals / Lookbook / Style Hub
3. **إعادة تسمية** صفحة Shop → Outfits
4. **نظام كوبونات** متكامل مع انتهاء صلاحية تلقائي
5. **إعادة هيكلة قاعدة البيانات** بالكامل مع جداول جديدة

### المرحلة 1 – إعادة هيكلة قاعدة البيانات

**الأولوية: أولاً (كل شيء يعتمد عليها)**

#### الجداول المطلوبة

| الجدول | الوضع | ملاحظات |
|--------|-------|---------|
| `genders` | جديد | `id, name (men/women/kids), slug` |
| `categories` | تعديل | إضافة `parent_id, gender_id, slug, sort_order` |
| `category_images` | جديد | صور Slider للـ Navbar لكل gender |
| `products` | تعديل | إضافة `discount_percent, views_count, is_deal` |
| `tags` | جديد | `id, name, slug` |
| `product_tags` | جديد | جدول ربط products ↔ tags |
| `reviews` | جديد | `id, product_id, user_id, rating, body, created_at` |
| `product_views` | جديد | تتبع مشاهدات المنتج |
| `wishlists` | تعديل/تأكيد | التأكد من الهيكل الحالي |
| `carts` | جديد | `id, user_id, product_id, quantity` |
| `coupons` | جديد | `id, code, store_name, discount_percent, start_date, end_date, affiliate_link, is_active` |
| `saved_coupons` | جديد | ربط users ↔ coupons |
| `deals` | جديد | علاقة منتجات بخصم لتمييزها كـ"عروض" |
| `saved_deals` | جديد | ربط users ↔ deals |
| `lookbooks` | جديد | `id, title, cover_image, created_at` |
| `lookbook_products` | جديد | ربط lookbooks ↔ products |

#### SQL الأساسي (ترتيب التنفيذ)

```sql
-- 1. Genders
CREATE TABLE genders (
  id   SERIAL PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,  -- 'men' | 'women' | 'kids'
  slug TEXT NOT NULL UNIQUE
);

-- 2. Categories (hierarchical)
ALTER TABLE categories
  ADD COLUMN parent_id   INT REFERENCES categories(id),
  ADD COLUMN gender_id   INT REFERENCES genders(id),
  ADD COLUMN slug        TEXT UNIQUE,
  ADD COLUMN sort_order  INT DEFAULT 0;

-- 3. Category images (navbar slider)
CREATE TABLE category_images (
  id          SERIAL PRIMARY KEY,
  gender_id   INT NOT NULL REFERENCES genders(id),
  image_url   TEXT NOT NULL,
  alt_text    TEXT,
  sort_order  INT DEFAULT 0
);

-- 4. Tags
CREATE TABLE tags (
  id   SERIAL PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,
  slug TEXT NOT NULL UNIQUE
);

CREATE TABLE product_tags (
  product_id INT REFERENCES products(id) ON DELETE CASCADE,
  tag_id     INT REFERENCES tags(id)     ON DELETE CASCADE,
  PRIMARY KEY (product_id, tag_id)
);

-- 5. Reviews
CREATE TABLE reviews (
  id         SERIAL PRIMARY KEY,
  product_id INT  NOT NULL REFERENCES products(id) ON DELETE CASCADE,
  user_id    UUID NOT NULL REFERENCES auth.users(id),
  rating     SMALLINT CHECK (rating BETWEEN 1 AND 5),
  body       TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 6. Product views
CREATE TABLE product_views (
  id         SERIAL PRIMARY KEY,
  product_id INT  NOT NULL REFERENCES products(id) ON DELETE CASCADE,
  user_id    UUID REFERENCES auth.users(id),
  viewed_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 7. Carts
CREATE TABLE carts (
  id         SERIAL PRIMARY KEY,
  user_id    UUID NOT NULL REFERENCES auth.users(id),
  product_id INT  NOT NULL REFERENCES products(id),
  quantity   INT  NOT NULL DEFAULT 1,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE (user_id, product_id)
);

-- 8. Coupons
CREATE TABLE coupons (
  id               SERIAL PRIMARY KEY,
  code             TEXT NOT NULL UNIQUE,
  store_name       TEXT NOT NULL,
  discount_percent SMALLINT CHECK (discount_percent BETWEEN 1 AND 100),
  start_date       TIMESTAMPTZ NOT NULL,
  end_date         TIMESTAMPTZ NOT NULL,
  affiliate_link   TEXT,
  is_active        BOOLEAN DEFAULT TRUE,
  image_url        TEXT,
  created_at       TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE saved_coupons (
  user_id   UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  coupon_id INT  REFERENCES coupons(id)    ON DELETE CASCADE,
  PRIMARY KEY (user_id, coupon_id)
);

-- 9. Saved Deals
CREATE TABLE saved_deals (
  user_id    UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  product_id INT  REFERENCES products(id)   ON DELETE CASCADE,
  PRIMARY KEY (user_id, product_id)
);

-- 10. Lookbooks
CREATE TABLE lookbooks (
  id          SERIAL PRIMARY KEY,
  title       TEXT NOT NULL,
  cover_image TEXT,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE lookbook_products (
  lookbook_id INT REFERENCES lookbooks(id)  ON DELETE CASCADE,
  product_id  INT REFERENCES products(id)   ON DELETE CASCADE,
  PRIMARY KEY (lookbook_id, product_id)
);

-- Indexes
CREATE INDEX idx_categories_gender    ON categories(gender_id);
CREATE INDEX idx_categories_parent    ON categories(parent_id);
CREATE INDEX idx_products_category    ON products(category_id);
CREATE INDEX idx_reviews_product      ON reviews(product_id);
CREATE INDEX idx_product_tags_product ON product_tags(product_id);
CREATE INDEX idx_coupons_active_date  ON coupons(is_active, end_date);
```

#### RLS Policies المطلوبة

- **categories / genders / tags**: `public read`, `admin write`
- **reviews**: `public read`, `authenticated insert` (1 review per user per product)
- **coupons**: `public read`, `admin write`
- **saved_coupons / saved_deals / wishlists / carts**: `owner-only read+write`

### المرحلة 3 – نظام الفلترة الموحد (Unified Filtering Engine)

**⚠️ ملاحظة هامة:** نظام الفلترة في Outfits موجود بالفعل. المطلوب فقط التأكد من التوافق مع الهيكل الجديد.

#### طبقة الخدمة المركزية

```
src/services/
└── filterService.ts   ← نقطة الدخول الوحيدة لكل الفلترة
```

#### Query Builder Logic

```typescript
interface FilterParams {
  gender?:      string;
  categoryId?:  number;
  subCategory?: number;
  tags?:        string[];
  colors?:      string[];
  priceMin?:    number;
  priceMax?:    number;
  brand?:       string;
  hasDiscount?: boolean;
}

function buildFilterQuery(params: FilterParams) {
  let query = supabase.from('products').select('...');
  if (params.gender)      query = query.eq('gender_id', ...);
  if (params.categoryId)  query = query.eq('category_id', ...);
  if (params.hasDiscount) query = query.gt('discount_percent', 0);
  // ...
  return query;
}
```

#### URL State Strategy

```
/outfits?gender=men&category=tops&sub=t-shirts&color=black&price=0-500
```

- استخدام `useSearchParams()` من Next.js
- `nuqs` library للمزامنة السهلة (اختياري)
- Server-side filtering عبر `searchParams` في Server Components

### المرحلة 6 – Navbar Mega Menu

#### الهيكل البصري (Desktop)

```
┌──────────────────────────────────────────────────────┐
│  [View All Men ↗]                                    │
├─────────────────────┬────────────────────────────────┤
│  Tops               │  ┌──────────────────────────┐  │
│    └ T-Shirts       │  │                          │  │
│    └ Shirts         │  │   Image Slider           │  │
│    └ Hoodies        │  │   (2-3 images, auto)     │  │
│    └ Jackets        │  │                          │  │
│  Bottoms            │  └──────────────────────────┘  │
│    └ Pants          │                                │
│    └ Jeans          │                                │
│  Footwear           │                                │
│  Accessories        │                                │
│  Perfumes           │                                │
└─────────────────────┴────────────────────────────────┘
```

#### Animation Spec

| الخاصية | القيمة |
|---------|-------|
| Animation type | `opacity: 0→1` + `translateY: -8px→0` |
| Duration | `200ms` |
| Easing | `cubic-bezier(0.16, 1, 0.3, 1)` |
| Open delay | `150ms` |
| Close delay | `200ms` |
| GPU rendering | `will-change: transform, opacity` |

#### Hover State Machine

```
IDLE
  → (hover navitem, 150ms delay) → OPENING
  → OPEN
  → (mouse leaves, 200ms grace) → CLOSING
  → IDLE

State: { activeMenu: 'men' | 'women' | 'kids' | null, timer: NodeJS.Timeout }
```

#### Mobile Behavior (Accordion)

- Click → expand accordion (collapsing others)
- Scroll lock عبر `document.body.style.overflow = 'hidden'`
- Slider يظهر أسفل "View All"
- Touch-optimized (min 44px tap targets)

#### Accessibility

```html
<nav role="navigation" aria-label="Main navigation">
  <button aria-expanded="true" aria-controls="men-dropdown">Men</button>
  <div id="men-dropdown" role="region" aria-label="Men categories">
    ...
  </div>
</nav>
```

- `Escape` → يغلق القائمة
- Tab navigation عبر القائمة
- Focus يعود لـ trigger عند الإغلاق

#### Slider Spec

- 2-3 صور من `category_images` (DB-driven)
- Auto-slide كل `3500ms`
- `loading="lazy"` على كل صورة
- CSS transitions فقط (لا JS animation)
- Pause on hover

### المرحلة 7 – Performance Optimization

#### Strategy

| الموضوع | الحل |
|---------|------|
| Category tree | `React.cache()` + `useMemo` |
| Navbar re-render | `memo` على subcomponents |
| Images | `next/image` مع `loading="lazy"` |
| Animations | `transform` + `opacity` فقط |
| Filtering | Server-side عبر `searchParams` |
| Pagination | Cursor-based (infinite scroll) |
| Queries | `select` محددة (لا `*`) |

#### Caching Strategy

```typescript
// Server Component caching
const categories = await cache(getCategoryTree)(); // خلال الـ request

// Client-side
const store = useMemo(() => buildCategoryMap(categories), [categories]);
```

### المرحلة 8 – Security & Testing

#### RLS Plans

```sql
-- Public read for categories
CREATE POLICY "public read categories"
  ON categories FOR SELECT USING (true);

-- Owner-only for saved items
CREATE POLICY "owner saved_deals"
  ON saved_deals FOR ALL
  USING (auth.uid() = user_id);

-- Admin-only writes
CREATE POLICY "admin write coupons"
  ON coupons FOR INSERT
  USING (EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND is_admin = true));
```

## ⚠️ تحذيرات مهمة

> [!IMPORTANT]
> نظام الفلترة في صفحة Outfits موجود مسبقاً. لا تعيد بناءه من الصفر. فقط تأكد من توافقه مع الهيكل الجديد للفئات.

> [!WARNING]  
> عند تعديل جدول `categories`، يجب الحذر من البيانات الموجودة مسبقاً. استخدم `ALTER TABLE` وليس `DROP + CREATE`.

> [!WARNING]
> عند إعادة تسمية `/shop` → `/outfits`، أضف redirect في `next.config.js` لتجنب broken links.

> [!NOTE]
> جميع الصور يجب أن تكون `.webp` وبحجم أقصى 500KB قبل الرفع على Supabase Storage.
