# TASK 1 — Product Rendering Issue

## Problem
When entering the products page, products appear normally.  
However, after clicking on a product and returning back to the products page, the products disappear.

## Expected Behavior
- Products should appear instantly when navigating back.
- The following should persist:
  - Product list
  - Filters
  - Scroll position

## Root Cause

Possible causes:

1. **State Reset**
   Products are stored in `useShopFilters` using local state.  
   When navigating to a product page and returning, the component remounts and the state resets.

2. **No Caching**
   No caching mechanism exists, so every mount triggers a fresh fetch.

3. **Scroll Restoration**
   The browser may not restore the previous scroll position.

4. **Filter State Loss**
   URL query parameters containing filters may be lost during navigation.

## Solution

Use **TanStack Query (React Query)** for product fetching and caching.

### Implementation

Replace `useEffect + useState` with `useQuery`.

```ts
staleTime: 5 * 60 * 1000
````

### Navigation Improvement

Use:

```ts
router.push(url, { scroll: false })
```

### Optional Enhancement

Store fetched products in:

* React Context
* Redux

so they persist across navigation.

## Expected Result

* Products appear instantly when returning to the page.
* No loading flicker.
* Filters and scroll position remain intact.

# TASK 3 — Smart Global Live Search

## Goal

Implement a **Live Search System** across all content types:

* Products
* Lookbooks
* Coupons

## Search UI

Located inside the **Navbar**.

### Behavior

When the user clicks or focuses the search input:

* A dropdown or overlay appears.

### Search Result Groups

#### 🏷 Products

Display:

* Product image
* Product name
* Price
* Brand tag
* Piece type tag

#### 📚 Lookbooks

Display:

* Cover image
* Lookbook title

#### 🎟 Coupons

Display:

* Store logo
* Coupon code
* Discount badge

### Result Limits

Inside the dropdown:

* 3–5 results per section
* Include a **"See All"** link for each section.

---

## UX Logic

### Debounce

```
300ms
```

### Minimum Query Length

```
2 characters
```

### Keyboard Navigation

* Arrow keys → navigate results
* Enter → select result
* ESC → close dropdown

### UI States

* **Loading** → skeleton shimmer
* **Empty** → "No results found for '{query}'"

---

## API Implementation

Create a new query file:

```
src/lib/queries/search.ts
```

Function:

```
globalSearch(query: string)
```

### Parallel Queries

Execute three parallel Supabase queries.

#### Products

```
ilike("name", %query%)
```

Limit: 5 results

#### Lookbooks

```
ilike("name", %query%)
```

Limit: 5 results

#### Coupons

```
ilike("store_name", %query%)
OR
ilike("code", %query%)
```

Limit: 5 results

---

## Performance

* Debounce search input (300ms)
* Execute queries in parallel
* Limit results to reduce payload
* Cache recent searches in `sessionStorage`

# TASK 6 — Hover Image Switching in MegaMenu

## Problem

In the MegaMenu, the same images appear for all groups:

* Clothing
* Footwear
* Accessories
* Fragrances

Images should change dynamically depending on the hovered group.

---

## Expected Behavior

When hovering over:

```
Clothing
```

show clothing-related images.

When hovering over:

```
Footwear
```

show footwear-related images.

---

## Implementation

Add a state variable:

```
hoveredGroup
```

### Default State

Display all images.

### On Hover

Filter images by:

```
piece_type_group
```

### Animation

Use transition:

```
300ms crossfade
```

### Debounce

```
150ms
```

---

## Database Change

Add a new column to the `category_images` table:

```sql
ALTER TABLE category_images
ADD COLUMN piece_type_group varchar;
```

### Expected Values

```
clothing
footwear
accessories
fragrances
```
